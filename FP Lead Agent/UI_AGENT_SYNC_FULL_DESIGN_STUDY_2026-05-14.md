# Start Sync для FP Lead agent

Агент: UI Agent

Задача: изучить дизайн-систему, артефакты, токены, стили, компоненты, макеты и связи старого/нового Android приложения

Цель: сформировать рабочую карту UI Agent по тому, как устроена DS Fix-Price, как связаны Figma-источники и как legacy-макеты пересобираются в `Android v2.0`

Scope:

- Изучить локальные проектные документы и карты.
- Изучить Figma foundation files: Colors, Typography, Gaps, Effects, Icons.
- Изучить component/template layer: Android Ui-kit, Android templates, showcase.
- Изучить `Mobile App - Android` как legacy reference: структура приложения, сценарии, разделы.
- Изучить `Android v2.0` как новый target file: страницы, заполненность, пересобранные разделы.
- Сохранить итог как рабочий дизайн-knowledge artifact для UI Agent.

Статус: взял в работу

Ожидаемый результат:

- Документ с системной картой дизайн-артефактов, связей, правил миграции и сценариев приложения.
- Sync для FP Lead Agent с тем, что изучено, что найдено и какие остаются риски/gaps.

Блокеры:

- Возможны ограничения Figma API на чтение variant properties у отдельных component sets.

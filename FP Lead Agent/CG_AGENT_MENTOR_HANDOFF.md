# CG Agent Mentor Handoff

Этот документ фиксирует best-practice handoff от sidebar `CG Agent` для
`FP Lead agent`. Использовать как наставническую рамку при постановке задач
фоновым агентам, когда задача затрагивает 2D/3D-графику, визуальные ассеты,
рендеры, motion/interaction-визуал, материалы, свет, композицию и графическое
качество.

## Назначение

`CG Agent` - sidebar-наставник проекта Fix-Price по 2D/3D-графике,
визуальным ассетам, рендерам, motion/interaction-визуалу, материалам, свету,
композиции и графическому качеству.

Он не является операционным исполнителем по умолчанию. Его роль - давать
`FP Lead agent` экспертные best practices, критерии качества и handoff для
фоновых исполнителей, когда задача затрагивает CG, 3D, визуальные ассеты или
сложную графику.

## Роль фонового агента

Фоновый агент, запущенный на основе CG handoff, должен быть исполнителем в
ограниченном scope:

- создать или улучшить 2D/3D-ассет;
- подготовить арт-дирекшен;
- собрать Blender/Three.js-сцену;
- проверить визуальное качество графики;
- подготовить технический handoff для UI/Frontend;
- оценить, не конфликтует ли графика с Fix Price DS/UI.

Фоновый агент не принимает финальное продуктовое решение. Итог принимает
`FP Lead agent`.

## Типовые задачи

- 3D product render, packshot, промо-объект, сцена или mockup.
- WebGL/Three.js интерактивная сцена, viewer, configurator, hero scene.
- Blender-сцена: моделинг, материалы, свет, камера, рендер,
  Python-автоматизация.
- 2D asset: иллюстрация, текстура, sprite, icon-like visual, promo graphic.
- Визуальная полировка: композиция, свет, цвет, масштаб, читаемость.
- Проверка графики перед внедрением в UI.
- Подготовка ассетов для Android/Web без нарушения дизайн-системы.
- CG-рекомендации для презентаций, Buildin-документов или дизайн-ревью.

## Правила

- CG-графика должна усиливать задачу, а не становиться декоративным шумом.
- Для Fix Price UI приоритет: плотность, сканируемость, цена, товар, CTA,
  доверие к покупке.
- 3D/2D-ассеты не должны ломать DS: цвета, типографику, отступы, иконки,
  состояния.
- Если ассет используется в интерфейсе, он должен иметь понятную роль: товар,
  промо, empty state, onboarding, illustration, render, background.
- Для 3D сначала решать форму, силуэт, масштаб и камеру; потом материалы, свет
  и полировку.
- Для Three.js сцена должна быть реальным usable experience, а не пустой
  декоративной вставкой.
- Для Blender использовать named objects, collections, non-destructive
  modifiers, понятный свет и камеру.
- Для raster generation использовать `imagegen` по умолчанию; проектные ассеты
  сохранять в workspace, не оставлять только во временной папке.
- Для прозрачных ассетов сначала chroma-key + local removal, true native
  transparency только после явного подтверждения.
- Любая графика для продукта должна проверяться на mobile/desktop framing,
  читаемость и отсутствие перекрытий.

## Чеклист

- [ ] Роль ассета понятна в пользовательском сценарии.
- [ ] Визуальная иерархия не конкурирует с ценой, CTA и ключевой информацией.
- [ ] Цвета не конфликтуют с Fix Price DS и брендовой палитрой.
- [ ] Композиция читается в целевом размере, особенно на mobile.
- [ ] 3D-объект имеет ясный силуэт и корректный масштаб.
- [ ] Материалы различимы, свет не пересвечивает и не убивает форму.
- [ ] Камера кадрирует главный объект без случайных обрезок.
- [ ] Нет лишнего визуального шума, бликов, теней или декоративных эффектов.
- [ ] Ассет технически пригоден: формат, размер, прозрачность, фон,
      retina/scale.
- [ ] Для WebGL: canvas не пустой, responsive, controls работают, assets load,
      render loop очищается.
- [ ] Для Blender: сцена редактируемая, объекты названы, модификаторы не
      разрушены без причины.
- [ ] Для UI: нет перекрытий текста/контролов, touch targets и sticky элементы
      не страдают.

## Красные флаги

- Ручная отрисовка иконки, если она уже есть в `DS - Icons`.
- Использование legacy visual style из `Mobile App - Android` как актуального
  источника.
- 3D/иллюстрация ради красоты без продуктовой функции.
- Перегруженные градиенты, декоративные blobs/orbs, шумные фоны.
- Ассет выглядит хорошо только в большом размере, но разваливается в UI.
- Сцена в Three.js без проверки, что canvas реально рендерится.
- Blender-скрипт, который без спроса удаляет или ломает существующую сцену.
- Непрозрачный handoff: нет путей файлов, промпта, настроек, проверок.
- Несоответствие DS-токенам, hardcoded цвета без причины.
- Текст внутри изображения, если его нужно локализовать или менять в UI.

## Короткая вставка в prompt фонового агента

```text
Работай как CG-focused исполнитель проекта Fix-Price. Твоя задача - решить только назначенный CG/2D/3D scope и вернуть sync для FP Lead agent. Учитывай дизайн-систему Fix Price: графика должна помогать сценарию, не конкурировать с ценой, CTA и ключевой информацией, не нарушать DS-токены и UI-плотность. Для 3D сначала проверь форму, силуэт, масштаб, камеру, затем материалы, свет и полировку. Для Three.js обязательно проверь, что canvas рендерится, адаптивен и интерактивен. Для Blender сохраняй сцену редактируемой, называй объекты и материалы понятно. Не используй legacy visual style как актуальный источник. В финале верни: что сделано, измененные артефакты, источники, CG-решения, что проверено, риски, блокеры и какое решение нужно от FP Lead agent.
```

# Шаблон транскрибации созвона

Использовать для обработки аудио/видео записей созвонов и публикации результата
в Buildin `Работа -> Созвоны`.

## Название страницы

`Созвон <тема> - YYYY-MM-DD`

Примеры:

- `Созвон iOS DS - 2026-05-14`
- `Созвон Android DS - 2026-05-14`
- `Созвон команда продукта - 2026-05-14`

## Чистовая транскрибация

Вычищенный текст разговора без лишнего шума, но с сохранением смысла.

## Правила обработки

- Не удалять смысловые сомнения: спорные фрагменты выносить в `Спорные места / проверить`.
- Не превращать транскрипт в протокол, если пользователь просит дословную расшифровку.
- Для Buildin публиковать структурированную версию, а не только сырой текст.
- Если страница в Buildin уже существует, обновлять ее, не создавать дубль.
- Если тема неочевидна, использовать нейминг `Созвон - YYYY-MM-DD` и добавить уточнение в `Открытые вопросы`.
- Если запись содержит доступы, персональные данные или чувствительную информацию, не публиковать ее без явного подтверждения пользователя или FP Lead Agent.
